# ETAP 6 — QA CHECKLIST (FRONT)

## Testy ręczne
- Start dev: `npm install && npm run dev` w `frontend/`, sprawdź `/login` i logowanie tokenem.
- Czat SSE: wyślij wiadomość, potwierdź stream tokenów i finalny done, brak freeze UI.
- Nowa sesja: wejście na `/app/chat`, wysłanie pierwszej wiadomości tworzy sesję i pojawia się w sidebarze.
- Historia: kliknięcie istniejącej sesji ładuje poprzednie wiadomości.
- Upload: dodaj plik, sprawdź chip i brak błędów (wymaga poprawnego AUTH_TOKEN i UPLOAD_DIR na backendzie).
- Ustawienia: zmiana tokenu/API base zapisuje się i jest używana do kolejnych requestów.
- Health/Audio status: panel prawy pokazuje status `/health`, TTS/STT (w zależności od kluczy środowiskowych backendu).

## Scenariusze awaryjne
- SSE drop: użyj "Zatrzymaj" (AbortController) i wyślij ponownie wiadomość.
- 401: backend zwraca 401 => token czyszczony, przekierowanie do `/login`.
- 413 upload: backend zwraca błąd wielkości — alert z komunikatem.

## Performance
- Lista wiadomości do ~200 wpisów działa bez wirtualizacji (limit zapytań backendu). Przy większych wolumenach rozważyć react-virtualized.
